# Zenn Article Planner

Zennなど技術ブログ向けに、Obsidianのアイデアメモから記事構成案を生成するスキル。アイデア選定→記事タイプ判定→タイトル候補→読者設定→セクション構成の順で出力する。

## Step 1: アイデアの収集

以下のパスを読んでアイデアの一覧を把握する。

```
~/Obsidian/MyVault/Articles/Ideas/_inbox.md   # とりあえずメモ置き場
~/Obsidian/MyVault/Articles/Ideas/            # 個別ファイルがあれば全て読む
```

個別ノートがあれば、その内容（キーポイント・一言サマリー・関連リンク）も読んで理解を深める。

## Step 2: アイデアの提示と選定

読んだネタを一覧で表示し、ユーザーに選んでもらうか、最も「記事化しやすい」ネタを推薦する。

**記事化しやすい基準（優先度順）：**
1. 詰まりポイント＋解決策がある
2. 再現性がある（手順が追える）
3. 比較・選定の判断軸がある
4. 「なぜそうするか」の理由が説明できる

## Step 3: 記事タイプの判定

選んだネタに最も合う記事タイプを判定する。

| タイプ | 特徴 | 向いているネタ |
|--------|------|----------------|
| **チュートリアル** | 手順を追って完成まで導く | セットアップ・ハンズオン |
| **Tips・小ネタ** | 短く・すぐ使える知識 | ワンライナー・設定の工夫 |
| **比較・選定** | AとBを並べてどちらか選ぶ | ライブラリ選定・アーキテクチャ選択 |
| **ディープダイブ** | 仕組みの深掘り・内部解説 | 「なぜ動くのか」「何が起きているか」 |
| **環境構築・セットアップ** | 環境を整える手順 | 開発環境・ツール設定 |
| **課題解決レポート** | 問題→原因→解決のストーリー | バグ調査・パフォーマンス改善 |

## Step 4: 記事の構成案を生成

以下をすべて出力する。

### 4-1. タイトル候補（3〜5案）

Zennで伸びるタイトルの特徴を踏まえて複数案を出す：
- 具体的な数値・バージョンを入れる（例: 「5分で」「3つの方法」）
- 読者のメリットが一目でわかる
- 検索されやすいキーワードを含む
- 問いかけ型・結論先出し型など形式を変えて提案する

### 4-2. 読者設定

```
対象読者: （例: ReactをひととおりできるフロントエンドエンジニアでFlutterは未経験）
前提知識: （例: JavaScript・HTML/CSS の基礎）
難易度: 初級 / 中級 / 上級
読後の状態: （例: ○○を自分でセットアップして動かせるようになる）
```

### 4-3. 記事の骨格（h2/h3構成）

各h2に対して「このセクションで伝えること」を1〜2文で説明する。

```markdown
## はじめに
（なぜこの記事を書いたか・誰に向けた記事か・読むと何ができるか）

## [本文セクション1] - h2タイトル
> このセクションで伝えること: ○○

### h3タイトル（必要な場合）

## [本文セクション2] - h2タイトル
> このセクションで伝えること: ○○

## まとめ
（やったことの振り返り・次のステップ・参考リンク）
```

### 4-4. Zenn向けの補足ヒント

記事タイプに応じて、Zennで読まれやすくするためのポイントを2〜3点添える。

例：
- チュートリアル → コードブロックに言語指定・コピーしやすいワンコマンド化
- 比較記事 → 比較表を最初に置くと離脱率が下がる
- ディープダイブ → 冒頭に「TL;DR」を置いて忙しい読者に配慮

## Step 5: 次のアクション提案

構成案を出したあと、以下のいずれかを提案する：

```
[A] このままZennのドラフトファイルを生成する（~/Desktop/zenn/ に npx zenn new:article）
[B] Obsidianに構成案を下書きとして保存する（Articles/Drafts/）
[C] タイトルや構成を修正してから進める
```

## Zennのファイル操作（Aを選んだ場合）

```bash
cd ~/Desktop/zenn
npx zenn new:article --slug <slug名> --title "<タイトル>" --type tech --emoji 🔧
```

生成したファイルに構成案のh2/h3見出しを書き込み、各セクションに「TODO: ここに書く内容」のコメントを入れる。

## 注意点

- `_inbox.md` のネタに「※ 執筆推奨でない」などのメモがある場合は除外して候補として提示しない
- Obsidianに同じテーマの既存ドラフトがある場合は重複を警告する
- 記事タイプが複合的な場合（例: 比較＋チュートリアル）は両方の特徴を組み合わせる
